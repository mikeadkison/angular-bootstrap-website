Mutespeak

I play a game called Team Fortress 2 competitively. In competitive play, players will forms teams and compete for one season (several months) against other teams. Because fast coordination and information-sharing is important, teams use VoIP programs such as Mumble to communicate with their voices during gameplay.

Mutespeak helps people without the ability to speak (no microphone, speech disability) to communicate quickly in VoIP programs. It allows the user to pair global key shortcuts of his choice to voice sayings which are generated on the fly with text-to-speech.

Usage

  1. Install 64 bit Java
  2. Download the mutespeak release
  3. Unzip tts.zip
  4. Run the .jar file
  5. Enter your key binds and their associated speech sayings
  6. Use PulseAudio or Audio Router with Virtual Audio Cable or just useWindows Stereo Mix to route the output of the program to a virtual microphone.
  7. In your VoIP program, select the virtual mic as your input
  8. Start up your game
  9. Press the key binds corresponding to what you would like the voice to say. You may also need to press your VoIP program's Push-to-talk key at the same time.

Limitations

The program works on Windows only. The reason for this is that it is hard to find good (understandable) cross-platform text-to-speech libraries, and also hard to find good cross-platform global key bind libraries because global key binds are operating system-specific in their implementation. Since most people game on Windows, and the person I originally made this for uses Windows, I wrote this program for Windows.

Info

Mutespeak uses Windows's built-in TTS by generating a VBS script on the fly which is able to use the Microsoft Speech API. This was the easiest way to get TTS on Windows. The alternatives would require including additional .jars, and many alternatives sound bad or cost money.

A python version of mutespeak made by someone else which makes use of espeak is available in an unfinished state here. Since espeak is available on linux, this version might make a good linux-compatible mutespeak if someone takes the time to finish it. Warning: espeak sounds awful.
